11 ### I have an issue with GNOME keyring
12 
13 * Disable GNOME keyring
14 * Create a `~/.gnupg/gpg-agent.conf` containing:
15 
16 ```
17 enable-ssh-support
18 write-env-file
19 use-standard-socket
20 default-cache-ttl 600
21 max-cache-ttl 7200
22 ```
23 
24 Also, the following is useful to add to
25 your .bashrc if you are using Yubikey NEO on Ubuntu:
26 
27 ```
28 # OpenPGP applet support for YubiKey NEO
29 if [ ! -f /tmp/gpg-agent.env ]; then
30  killall gpg-agent;
31  eval $(gpg-agent --daemon --enable-ssh-support > /tmp/gpg-agent.env);
32 fi
33 . /tmp/gpg-agent.env
34 ```
35 
36 * More info: [issue 60](https://github.com/IJHack/qtpass/issues/60) and [issue 73](https://github.com/IJHack/qtpass/issues/73)

38 ### I don't get a passphrase / PIN dialog
39 
40 * You'll need to install pinentry-qt (or -qt4 or -qt5 or even -gtk) and
41  possibly set the full path to that executable in your `~/.gnupg/gpg-agent.conf`
42  for example: `pinentry-program /usr/bin/pinentry-qt4`
43 * On some esotheric systems it might be necessary to create a symbolic
44  link `/usr/bin/pinentry` to your pinentry application of choice
45  eg: `ln -s /usr/bin/pinentry-qt5 /usr/bin/pinentry`
46 * On MacOs `pinentry-program /usr/local/bin/pinentry-mac` works after installing `pinentry-mac` from homebrew.

49 ### I have an other issue with gpg
50 
51 * Possibly you have you key only in gpg and not in gpg2
52 
53 ```
54 gpg --export [ID] > public.key
55 gpg --export-secret-key [ID] > private.key
56 gpg2 --import public.key
57 gpg2 --import private.key
58 rm public.key private.key
59 ```
60 Where [ID] is your gpg key-id.
61 * It might be the case where it is the other way around, exchange gpg and gpg2 accordingly . .

63 ### Git doesn't work on Windows
64 
65 git for Windows comes with an `ssh-askpass` compatible command, git `gui--askpass` (located in `/mingw64/libexec/git-core/git-gui--askpass` on PortableGit version, presumably some place similar for the installed version).

72 ### Where is the configuration stored?
73 
74 QtPass tries to use the native config choice for the OS it's running.
75 
76 * Linux and BSD: `$HOME/.config/IJHack/QtPass.conf`
77 * macOS: `$HOME/Library/Preferences/com.IJHack.QtPass.plist`
78 * Windows registry: `HKEY_CURRENT_USER\Software\IJhack\QtPass`
79 
80 These settings can be over-ruled by a `qtpass.ini` file in the folder where the application resides.
81 So called "portable config".
82 
83 There are some things to take care of when trying to sync on some systems (especially OSX, with regards to text and binary .plist files).
84 
85 More information: http://doc.qt.io/qt-5/qsettings.html#platform-specific-notes

102 ### I get icons that do not fit my (X11) default
103 
104 * On some WindowManagers, Qt doesn't know what icon set to use. A trick:
105 ```
106 export DESKTOP_SESSION=gnome
107 ```
108 
109 * Another possible reason is, that the currently installed Qt Version gives problems (e.g. on Linux Mint 17.3)
110 Then you'll have to install the current version via your package manager or if this is not up-to-date,
111 download it from https://www.qt.io/download/ install it and run:
112 ```
113 /PATHTOYOURQTINSTALLATION/5.5/gcc_64/bin/qmake
114 make
115 (sudo) make install
116 ```
117 where `PATHTOYOURINSTALLATION` is the path you selected in the qt installer (default `/home/YOURUSER/Qt/` )
118 and 5.5 has to be adapted for the Qt version you downloaded.

125 ### QtPass is not in my native language
126

130 ### How do I set the language manually?
131 
132 QtPass uses the system language. Changing it depends on your system:
133 * on Linux: ```LANGUAGE=fr qtpass``` will run QtPass in French.
