5 ### Can't save a password
6 
7 * Is folder initialised? Easiest way is to use the [Users] button

11 ### I have an issue with GNOME keyring
12 
13 * Disable GNOME keyring
14 * Create a `~/.gnupg/gpg-agent.conf` containing:
15 
16 ```
17 enable-ssh-support
18 write-env-file
19 use-standard-socket
20 default-cache-ttl 600
21 max-cache-ttl 7200
22 ```
23 
24 Also, the following is useful to add to
25 your .bashrc if you are using Yubikey NEO on Ubuntu:
26 
27 ```
28 # OpenPGP applet support for YubiKey NEO
29 if [ ! -f /tmp/gpg-agent.env ]; then
30  killall gpg-agent;
31  eval $(gpg-agent --daemon --enable-ssh-support > /tmp/gpg-agent.env);
32 fi
33 . /tmp/gpg-agent.env
34 ```
35 
36 * More info: [issue 60](https://github.com/IJHack/qtpass/issues/60) and [issue 73](https://github.com/IJHack/qtpass/issues/73)

38 ### I don't get a passphrase / PIN dialog
39 
40 * You'll need to install pinentry-qt (or -qt4 or -qt5 or even -gtk) and
41  possibly set the full path to that executable in your `~/.gnupg/gpg-agent.conf`
42  for example: `pinentry-program /usr/bin/pinentry-qt4`
43 * On some esotheric systems it might be necessary to create a symbolic
44  link `/usr/bin/pinentry` to your pinentry application of choice
45  eg: `ln -s /usr/bin/pinentry-qt5 /usr/bin/pinentry`

47 ### Git doesn't work on Windows
48 
49 git for Windows comes with an `ssh-askpass` compatible command, git `gui--askpass` (located in `/mingw64/libexec/git-core/git-gui--askpass` on PortableGit version, presumably some place similar for the installed version).

51 ### Where is the configuration stored?
52 
53 QtPass tries to use the native config choice for the OS it's running.
54 
55 * Linux and BSD: `$HOME/.config/IJHack/QtPass.conf`
56 * macOS: `$HOME/Library/Preferences/com.IJHack.QtPass.plist`
57 * Windows registry: `HKEY_CURRENT_USER\Software\IJhack\QtPass`
58 
59 These settings can be over-ruled by a `qtpass.ini` file in the folder where the application resides.
60 So called "portable config".
61 
62 There are some things to take care of when trying to sync on some systems (especially OSX, with regards to text and binary .plist files).
63 
64 More information: http://doc.qt.io/qt-5/qsettings.html#platform-specific-notes

81 ### I get icons that do not fit my (X11) default
82 
83 * On some WindowManagers, Qt doesn't know what icon set to use. A trick:
84 ```
85 export DESKTOP_SESSION=gnome
86 ```
87 
88 * Another possible reason is, that the currently installed Qt Version gives problems (e.g. on Linux Mint 17.3)
89 Then you'll have to install the current version via your package manager or if this is not up-to-date,
90 download it from https://www.qt.io/download/ install it and run:
91 ```
92 /PATHTOYOURQTINSTALLATION/5.5/gcc_64/bin/qmake
93 make
94 (sudo) make install
95 ```
96 where `PATHTOYOURINSTALLATION` is the path you selected in the qt installer (default `/home/YOURUSER/Qt/` )
97 and 5.5 has to be adapted for the Qt version you downloaded.
